REPOSITORY STRUCTURE AND OUTPUTS
================================

This file explains the cleaned structure of the repository after the
uncertainty-aware, target-free workflow update and final root reorganization.

FOLDER MAP (root has no loose files)
-------------------------------------
qsar_core/               Core Python modules (data_loader, descriptors, model,
                           clustering, visualization, paths)
scripts/                 Standalone runnable scripts (main.py, run_pipeline.py,
                           debug_main.py, all generate_* scripts, test scripts)
config/                  Environment configuration (config/environment.yml,
                           config/requirements.txt, config/setup_env.sh)
data/raw/                Raw input dataset (TB Project QSAR.xlsx)
reinvent_integration/    Generation, scoring, and target-free ranking workflow
notebooks/               Jupyter notebooks (canonical copies)
outputs/                 Ranked candidate outputs, generated CSVs, summaries
docs/                    Project documentation (this file and others)
archive/                 Archived/legacy files moved out of the active repo root
presentation/            LaTeX slide deck assets
external_tools/          External or vendor-specific integrations

HOW TO RUN (from repo root)
-----------------------------
# Export/rebuild the QSAR artifact:
  python reinvent_integration/export_qsar_model.py

# Train QSAR and run the full pipeline:
  python scripts/run_pipeline.py

# Rebuild Gate 1/2/3 ranked outputs:
  python reinvent_integration/run_hybrid_stage_gates.py

# Quick import smoke-test:
  python scripts/test_import.py

# Dataset check:
  python scripts/test_data.py

1. SOURCE CODE
--------------
qsar_core/
  Core project modules. Imported as `from qsar_core.X import Y`.

  Files:
  - data_loader.py      — Excel loading, SMILES cleaning, deduplication
  - descriptors.py      — RDKit, ECFP, MACCS, Mordred descriptor generators
  - model.py            — Model factory, train_and_select, uncertainty helpers
  - clustering.py       — UMAP, HDBSCAN, scaffold functions
  - visualization.py    — Plot helpers (IC50 distribution, UMAP, parity)
  - paths.py            — Shared PROJECT_ROOT and DATASET_XLSX constants
  - __init__.py         — Re-exports PROJECT_ROOT and DATASET_XLSX

reinvent_integration/
  Central workflow code.

  Key files:
  - export_qsar_model.py
      Builds the QSAR artifact and stores up to five positive-R² models for
      uncertainty estimation.
  - qsar_ensemble.py
      Computes ensemble predictions, standard deviation, and confidence bounds.
  - score_generated_smiles_adme.py
      Adds potency, ADME, alerts, synthetic accessibility, and QSAR uncertainty.
  - run_hybrid_stage_gates.py
      Produces the final ranked Gate 1, Gate 2, and Gate 3 outputs.

scripts/
  All standalone scripts that are run from the command line. Every script
  uses Path(__file__).resolve().parents[1] to locate the project root and
  imports from qsar_core.*, so they work regardless of working directory.

config/
  - config/environment.yml      — Conda/mamba environment specification
  - config/requirements.txt     — Pip requirements
  - config/setup_env.sh         — Convenience shell script to create the conda env

data/
  - raw/TB Project QSAR.xlsx   — Primary input dataset
